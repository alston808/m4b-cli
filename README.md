# m4b-cli

A zero-dependency command-line utility for automated audiobook merging, high-quality encoding, and precision metadata tagging. 

Designed to operate securely and efficiently, this script acts as a bridge for web-scraped audiobook payloads. It seamlessly digests both raw directories and `.zip` exports (such as those generated by LibbyRip), handling the extraction, cover art downloading, chapter mapping, and containerized audio processing autonomously.

## Features

* **Archive Support:** Pass a `.zip` file directly. The tool unpacks it to ephemeral storage, processes it, outputs the compiled `.m4b`, and safely deletes the raw stems.
* **Smart Chapter Mapping:** Converts relative JSON spine offsets into absolute `mp4chaps` timestamps required for proper playback navigation.
* **Auto-Resolution Cover Art:** Dynamically downloads the high-resolution cover image from the CDN if `metadata.json` includes a `coverUrl` but lacks a local image file.
* **High-Quality Audio:** Encodes output using `libfdk_aac` for optimal voice retention via transparent compression.
* **Dynamic Bitrate Detection:** Uses `ffprobe` to match the source bitrate, preventing degradation.
* **Precision Tagging:** Bypasses standard shell character limits by using `sandreas/tone` to safely bind extensive descriptions, cover art, and custom narrator tags directly into the M4B container.

## Prerequisites

This script relies entirely on standard Python 3 libraries, eliminating the need for `pip` installations. However, you must have the following system-level tools installed:

* **Python 3.12+**
* **Podman** (Required for secure, rootless container processing)
* **FFmpeg** (Specifically requires `ffprobe` in your system PATH)

## Installation

### macOS (Homebrew)
```bash
brew tap alston808/m4b
brew install m4b
```

### Debian / Ubuntu (APT)
Download the `.deb` release:
```bash
sudo apt install ./m4b_1.0.0_all.deb
```

### Manual Installation
```bash
git clone [https://github.com/alston808/m4b-cli.git](https://github.com/alston808/m4b-cli.git)
cd m4b-cli
chmod +x m4b
sudo cp m4b /usr/local/bin/m4b
```

## Usage

You can target a pre-extracted folder or pass a `.zip` directly. By default, the output `.m4b` is saved to your current working directory.

```bash
# Process an exported ZIP archive directly
m4b ~/Downloads/Audiobook_Export.zip

# Process a raw directory
m4b ~/Audiobooks/RawStems/

# Specify a custom output directory
m4b ~/Downloads/book.zip -o ~/Media/Audiobooks/
```
